Pain Point

选个大模型，要打开 12 个标签页

当前主流大模型厂商超过 12 家，每家的计费方式、Token 单价、上下文窗口都不一样，且经常调整。开发者在选型时信息极度分散，Token 消耗极快的场景下选错模型可能让成本翻数倍。

Solution

三源交叉验证，数据可信度透明可见

不依赖单一数据源。三个独立来源按 product_id 对齐做价格仲裁，5%/20% 阈值投票裁决，输出 confidence 标签让用户感知数据可信度。

LiteLLM JSON

  • 社区维护的静态 JSON
  • 覆盖 9/12 厂商
  • 无鉴权，可缓存
  • per_token 主源

OpenRouter API

  • REST API，无鉴权
  • 覆盖 8/12 厂商
  • 独有 benchmarks 元数据
  • 提供模型发布时间

官网 Scraper

  • OpenAI / Anthropic / DeepSeek
  • HTTP + HTML 解析
  • 价格偏离时官网裁决
  • subscription / coding_plan 唯一来源
仲裁规则：3 源一致 <5% → high · 2 源偏离 >20% → 采信官网 · 3 源互差 → 中位数
Features

面向开发者的选型工具集

从搜索筛选到价格波动告警，覆盖选型全流程。

⌖

多维度筛选

按地区、计费类型、模型能力（text/vision/audio）多选叠加筛选，实时模糊搜索厂商名与模型名。

↗

价格对比

表格视图 + 卡片视图切换，CNY/USD 一键换算，点表头排序，行展开查看上下文窗口与购买链接。

◉

厂商详情页

每家厂商独立路由 #/provider/{id}，展示该厂商全部产品，含 per_token / 订阅 / Coding Plan。

⚠

波动告警

单字段波动 >20% 警告，>50% 阻断落盘并回退旧数据。双通道告警：网页 stale 标记 + 飞书 webhook。

⌁

Coding Plan 抢购

独有 Coding Plan 维度，覆盖智谱、火山、OpenCode 等经常售罄的套餐，快速比价决策。

✦

数据新鲜度

首页展示「最近更新于 X 小时前」，每家厂商独立状态标记（ok / failed / stale），失败可见。
